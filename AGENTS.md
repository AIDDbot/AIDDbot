# AIDDbot

This is not a traditional application. There is no application source code to build or test. 

The deliverable are **agent skills** tailored to implement AI-Driven Development (AIDD) workflows.

## Skills

- Folder: `.agents/skills/`
- Inventory and when-to-use: [`.agents/skills/skills.catalog.md`](./.agents/skills/skills.catalog.md)
- SDLC coverage (build / maintain / refactor): [`.agents/skills/skills.lifecycle.md`](./.agents/skills/skills.lifecycle.md)
- Human-oriented workflow docs are in [`docs/`](./docs/)

## Editing skills

- Keep `SKILL.md` files actionable: Role, Task, Guardrails (always-on invariants), Context (Inputs, References, Glossary), Steps (Research, Plan, Implement the Output), Verification (checkboxes). Use [`.agents/skills/skillify/assets/skill.template.md`](./.agents/skills/skillify/assets/skill.template.md) as the canonical structure. Always create or fix skills via [`/skillify`](./.agents/skills/skillify/SKILL.md) — do not edit skills ad hoc.
- Put guides and long checklists in `references/` (`{tier}.guide.md`, `*.guidelines.md`); put templates in `assets/`. Link them from the skill (same skill folder only).
- Put paths, slugs, personality, git rules, and the spec status chain in the consumer `{Agents_File}` (generated by `/explore` from `explore/assets/AGENTS.template.md`); put skill-specific rules in that skill's `SKILL.md`.
- Commands live in `.agents/commands/`: a lean instruction `.md` plus a `.README.md` (prose) and a `.LEEME.md` (Spanish); keep the pair in sync when the command changes.
- After changing a skill or command, align the docs `/skillify` tracks:
  - its own `README.md` + `LEEME.md` — the skill's or command's prose pair
  - the root [README](./README.md)
  - [catalog](./.agents/skills/skills.catalog.md) and [lifecycle](./.agents/skills/skills.lifecycle.md)
  - [workflow](./docs/AIDD.workflow.md)
  - pipeline docs if behavior or paths changed.
